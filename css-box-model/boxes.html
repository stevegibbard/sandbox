Headings Are Block Elements

Paragraphs are blocks, too. However, <em> and <strong> elements are not. They are inline elements.

Block elements define the flow of the HTML document, while inline elements do not.

The above descriptions shows the following key points of box and inline elements:

  • By default block boxes appear below the previous block element.
  • Width of a block box is set based on the width of the parent container. In this case the body of the HTML document so the box stretches to the width of the browser.
  • By default the height of a block box is determined by the content it contains. Narrowing the browser window will make the heading split across lines.
  • Inline boxes don't change vertical spacing, they aren't used for manipulating layout they are used for styling within a block.
  • Inline box width is determined by the content it contains, not the width of the parent element.

Any html element can be set as a block box or an inline box by changing the display property in CSS. This property is useful when working with images (by default treated as inline blocks) and <a> list items which need to be treated as buttons for a menu.

Boxes are made up, from iside out, of:

  1. Content
  2. Padding (a background fills to cover content+padding but stops at the border)
  3. Border (Useful to add when debugging layout problems as it shows the size of content+padding and highlights the margins)
  4. Margin (the size outside of the border). Inline elements ignore top and bottom margins so vertical space cannot be changed on inline elements using margin. If you want an inline element to have a top and or bottom margin then you must change the display property to block.

Margins or Padding?

  • Padding expands any background, margins are transparent
  • Padding is part of the click area of an element whilst margins are not
  • Margins collapse vertically (the largest margin on two adjacent blocks takes precedence), whilst padding does not. Two paragraphs with top-marging 50px and bottom-margin 25px next to each other in html would have a margin of 50px between them (not 75px).

Ways to Prevent Margin Collapse?

  • Add an empty DIV with non-zero height (e.g. padding-top:1px) in-between the two consecutive blocks.
  • Define blocks so that they all either only use top-margin or bottom-margin padding.
  • Use padding rather than a margin to seperate blocks. This is not suitable in all cases (E.g when a border is in use or when a background is part of the block).
  • Flexbox layout doesn't have collapsing margins.

Generic Container Elements

Generic boxes, <div> and <span> are useful for styling a web page

Difference between Span and Div ?

  • Div is used for block level content
  • Span is used for inline elements

Sizing Boxes

By default a box will stretch to the size of the content (height) and browser width. To make a box a set width we define the property in CSS.

Height and Width define the size of the content of a box, padding and border are always added on top of the dimensions specified in CSS.

The CSS property box-sizing can be used to change how width / height and padding / borders interact. By default box-sizing is set to content-box, this lead to padding and border being added on top of the defined width. If you want the width specified to also incorporate padding and border then box-sizingf can be changed to border-box.

Aligning Boxes

You can align content and inline elements with text-align. Aligning boxes though requires one of three methods:

  • auto-margins - centre alignment
  • floats - alignment of left/right placement
  • flexbox - complete control of alignment

Auto-margins are applied when you add the CSS value auto to the left/right marging property. When the element in question has a width defined in CSS then an auto margin will make sure the box is aligned to the centre of the container that it belongs to. Button C below demonstrates this.

Resetting Default Browser Styles

Each browser has it's own set of default styles for margin and padding - it is best practice to reset your styling at the top of the CSS styles sheet so that each browser uses the same defaults. To do this add:

* {
margin: 0;
padding: 0;
box-sizing: border-box; }
